# 问题分析

一通搜索后可以发现是 stylus 的问题，但这个问题在 20 年已经修复，详见 #2538，并实装在了 @0.54.8 版本中，修复方法也很简单，给报警告的几个属性设置 null 为默认值即可

20 年已经修复的问题为何在 22 年又出现了，使用 npx cross-env NODE_OPTIONS="--trace-warnings" hexo s 进行追踪可以发现 [3]，是 nib 的依赖里使用了旧版本的 stylus ，如下图所示

其实不管是 hexo-renderer-stylus 还是 nib ，在相应 Git 仓库的 package.json 里已经提高了对 stylus 的版本要求，而在 npmJs 源上前者上次发版是 2 年前， stylus 的更新 cover 到了，后者则是 6 年前，显然 cover 不到
